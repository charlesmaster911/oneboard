## 📋 참고: 관련 프로세스 3종

### 주문 발주 (6단계)
| 단계 | 설명 | 담당자 |
|---|---|---|
| 1. 카페24 주문 확인 | 관리자 → 주문관리 → 신규주문 `결제완료` 필터링 | 이호혁 차장 |
| 2. 스마트스토어/쿠팡 주문 확인 | 각 플랫폼 접속, 오후 4시 기준 누락 확인 | 이호혁 차장 |
| 3. 주문 리스트 취합 | Google Sheets 오늘 날짜 시트 복사 | 이호혁 차장 |
| 4. 재고 확인 | 재고관리 시트와 비교, 부족 시 빨간색 표시 | 이호혁 차장 |
| 5. 발주서 작성·전송 | 양식 작성 후 이메일/카톡으로 제조사 발송 | 이호혁 차장 |
| 6. 주문 상태 변경 | 각 플랫폼 상태를 '상품 준비 중'으로 변경 | 이호혁 차장 |

|---|---|---|
| 1. 반품 요청 확인 | 카페24/스마트스토어에서 요청 확인, 사유 확인(단순/불량) | 이호혁 차장 |

| 4. 환불 처리 | 카페24/네이버페이 환불 요청 처리 | 이호혁 차장 |

## 🎯 CS 4단계 핵심 요약 (team-manual.html 반영용)

```
문의확인 → FAQ 기반 답변 → 시간·채널별 응대기록 → Slack #cs_alert 이슈 공유
```

- **담당자 통합:** CS 응대 주 담당 = 이호혁 차장 (광고는 권 과장)
- **기록 원칙:** 고객명 + 문의 유형 + 응대 시간 + 채널 모두 Google Sheets 응대기록 시트에 기록
- **Escalation:** 즉시 대응 필요 시 Slack `#cs_alert` + `@팀장` 멘션
